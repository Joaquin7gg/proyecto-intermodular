# Proveedor cloud

Para la implementación de la parte cloud del proyecto se ha seleccionado **Amazon Web Services (AWS)** como proveedor de computación en la nube.

AWS es una de las plataformas cloud más utilizadas a nivel mundial y ofrece una amplia variedad de servicios que permiten desplegar aplicaciones, gestionar redes, almacenar datos y controlar accesos de forma flexible y escalable. Su uso está muy extendido en entornos profesionales, lo que lo convierte en una opción adecuada para simular un escenario realista dentro de este proyecto.

La elección de AWS se basa también en que ha sido el proveedor trabajado durante el curso, lo que permite aplicar de forma práctica los conocimientos adquiridos en clase. A lo largo del desarrollo del proyecto se ha utilizado AWS Academy, facilitando el acceso a un entorno real sin necesidad de disponer de infraestructura propia. Esto ha permitido trabajar con conceptos clave como instancias virtuales, redes privadas en la nube o almacenamiento remoto, fundamentales en cualquier arquitectura cloud actual.

En el contexto del proyecto, la adopción de AWS se plantea como una extensión de la infraestructura local existente. El sistema principal del club continúa funcionando en un entorno on-premise, mientras que la nube se utiliza para complementar estos servicios, permitiendo realizar pruebas, desplegar servicios accesibles desde el exterior y analizar el comportamiento de soluciones cloud en un entorno controlado.

Entre las principales ventajas que ofrece AWS para este proyecto destacan:

- **Escalabilidad**, permitiendo aumentar o reducir recursos según las necesidades del sistema.
- **Modelo de pago por uso**, evitando inversiones iniciales elevadas en hardware.
- **Alta disponibilidad**, gracias a la infraestructura distribuida de AWS.
- **Seguridad**, mediante herramientas como control de accesos, redes privadas y configuración de reglas de tráfico.
- **Flexibilidad**, que facilita la integración con entornos locales y la evolución hacia arquitecturas híbridas.

Además, AWS proporciona servicios específicos que encajan directamente con las necesidades del proyecto, como máquinas virtuales (EC2), redes privadas (VPC) o almacenamiento en la nube, lo que permite construir una solución completa sin depender de infraestructura física adicional.

En conclusión, AWS se presenta como una solución adecuada para este proyecto, ya que permite complementar la infraestructura local existente, mejorar la accesibilidad de los servicios y sentar las bases para una posible migración futura hacia entornos cloud más completos.
