## Descripción del proyecto

Este proyecto intermodular reúne el trabajo realizado en distintos módulos de **1º ASIR** a partir de un mismo caso práctico: el diseño e implementación de la infraestructura tecnológica de un club de fútbol con necesidades similares a las de una pequeña o mediana empresa.

A lo largo del proyecto se desarrolla una propuesta completa que abarca la infraestructura de red, la base de datos, la organización del inventario tecnológico, la infraestructura hardware, la integración de servicios en la nube y la presentación profesional del trabajo realizado.

La idea principal ha sido construir una solución coherente y realista, en la que todos los módulos estén relacionados entre sí y aporten una parte concreta al resultado final.

---

## Objetivos

- Diseñar una infraestructura tecnológica adaptada a las necesidades del club
- Organizar la red mediante VLAN y servicios de red
- Modelar y documentar la base de datos del sistema
- Definir una propuesta de hardware equilibrada y escalable
- Representar y transformar información estructurada mediante XML
- Integrar servicios cloud dentro de una arquitectura híbrida
- Presentar el proyecto de forma técnica y también profesional

---

## Estructura del proyecto

El repositorio está organizado por módulos, de manera que cada carpeta recoge la documentación, esquemas, configuraciones, scripts o materiales asociados a una asignatura concreta.

Cada módulo desarrolla una parte específica del proyecto, pero todos comparten el mismo contexto: la creación de un entorno tecnológico funcional, seguro, organizado y escalable para el club.

---

## Módulos incluidos

### Planificación y Administración de Redes
Diseño e implementación de la infraestructura de red del club, incluyendo segmentación mediante VLAN, inter-VLAN routing, DHCP, red inalámbrica, ACL, SSH e impresoras de red.

### Gestión de Bases de Datos
Diseño del modelo entidad-relación y del modelo relacional para gestionar la infraestructura informática del club, así como scripts SQL de creación, inserción, consultas y permisos.

### Lenguaje de Marcas
Representación del inventario tecnológico mediante XML, validación con DTD y transformación a HTML mediante XSLT, integrando además el resultado dentro del entorno web del proyecto.

### Fundamentos del Hardware
Propuesta de infraestructura hardware adaptada a los distintos departamentos del club, incluyendo equipos de usuario, workstations, portátiles, almacenamiento y periféricos.

### Computación en la Nube
Documentación de una arquitectura híbrida que combina infraestructura local con servicios cloud desplegados en **Amazon Web Services (AWS)**.

### Itinerario Personal para la Empleabilidad I
Parte orientada al perfil profesional, donde se trabaja la presentación del proyecto, la reflexión personal, el portfolio y la orientación futura dentro del sector IT.

## Resultado global del proyecto

El resultado final es una propuesta tecnológica completa y conectada entre sí, en la que cada módulo aporta una parte esencial del sistema. De esta forma, el proyecto no se limita a resolver tareas aisladas, sino que plantea una visión global de cómo diseñar, documentar y presentar una infraestructura tecnológica realista dentro del ámbito de la administración de sistemas.

## Conclusión

Este proyecto intermodular ha permitido trabajar de forma conjunta contenidos de distintas asignaturas, relacionándolos dentro de un mismo caso práctico. Gracias a ello, el resultado es más completo, más coherente y también más cercano a una situación real, ya que combina red, bases de datos, hardware, documentación técnica, servicios cloud y presentación profesional en un único repositorio.
